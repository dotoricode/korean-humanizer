# 개인 금지어 / 선호어 템플릿

자주 쓰는 단어 규칙이 있다면 이 파일을 자기 워크플로우에 맞게 편집한다. humanizer는 카탈로그 패턴보다 **이 파일을 먼저** 적용한다.

> 매 세션 동일한 리스트를 쓰는 게 아니라면, **인라인 한 줄 입력**(아래 "사용법 — 인라인" 섹션)이 가장 빠르다. 이 파일은 영구 저장이 필요할 때 쓴다.

---

## 사용법

### 인라인 한 줄 (가장 빠름, 모든 환경 공용)

파일 없이 사용자 메시지 한 줄로 끝낸다.

```
/korean-humanizer 금지=활용,매우,다양한; 선호=유용하다→쓸만하다, 사실상→실제로; 유지=결과적으로

[다듬을 텍스트]
```

- 키 별칭: `금지`/`ban`/`avoid`, `선호`/`prefer`, `유지`/`keep`.
- 항목 구분: `,` 또는 `;`.
- 선호 매핑: `A→B` / `A->B` / `A => B` 동일.

### 자연어 한 줄

```
"활용", "매우" 빼고, "유용하다"는 "쓸만하다"로 바꿔서 다듬어줘:
[텍스트]
```

### Claude Code / OpenCode / Codex (파일 영구 저장)

이 파일을 그대로 두고 편집하면 된다. SKILL.md 가 자동으로 읽는다.

### Claude.ai

대화 시작 시 이 파일 내용을 첫 메시지로 붙여 넣거나, 위의 인라인/자연어 형식으로 매 메시지 지정.

### ChatGPT / Cursor / Gemini

`PROMPT.md` 시스템 프롬프트 끝에 아래 블록을 그대로 추가하거나, 매 메시지 인라인으로 지정.

---

## My personal list (편집해서 사용)

```yaml
금지:
  # 한 번이라도 보이면 자연스러운 대안으로 치환
  - "활용"
  - "매우"
  - "다양한"
  - "혁신적인"
  - "극대화"
  - "~의 관점에서"

선호:
  # 좌측 표현이 보이면 우측으로 우선 치환
  - "유용하다" → "쓸만하다"
  - "사실상" → "실제로"
  - "수행하다" → "하다"
  - "구현하다" → "만들다"

유지:
  # 자동 치환에서 제외할 단어 (의도적으로 쓰는 것)
  - "결과적으로"  # 데이터 분석 글에서는 의도적
  - "즉"          # 짧은 정리 글에서는 필수
```

---

## 작성 팁

- **금지** 리스트는 너무 많이 넣지 않는다. 정말 거슬리는 5~10개로 시작.
- **선호** 리스트는 1:1 매핑. 좌측 표현이 좌측 자리에 등장할 때만 치환된다.
- **유지** 리스트는 카탈로그가 자동으로 빼야 할 단어인데 본인은 의도적으로 쓰는 경우.
- 도메인별로 파일을 따로 둘 수도 있다 (예: `personal-list-marketing.md`, `personal-list-email.md`).

## 예시: 개발자 블로그용 personal list

```yaml
금지:
  - "활용"
  - "다양한"
  - "효과적인"
  - "체계적으로"
  - "~을 통해"

선호:
  - "사용하다" → "쓰다"
  - "구현" → "만들기"
  - "수행" → "하기"
  - "도입" → "쓰기"

유지:
  - "구현"  # 코드 맥락에서는 정확한 단어라 유지
```

## 예시: 비즈니스 이메일용 personal list

```yaml
금지:
  - "사료됩니다"
  - "보다 신중한"
  - "~함에 있어"

선호:
  - "검토하다" → "봐보다"   # 캐주얼 톤
  - "협의" → "이야기"

유지:
  - "감사합니다"
  - "수고하셨습니다"
```
