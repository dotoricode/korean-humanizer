# Sprint 4 — v1.0.0 Stabilization + 베타 + Polish

> v1.0 = 안정화 약속 + 외부 검증 + 마지막 polish. 사용자가 룰 갑자기 바뀔 걱정 없이 의존할 수 있는 상태.

| 항목 | 값 |
|---|---|
| 버전 | v0.8 → **v1.0.0** |
| 기간 | 3 주 (15 working days) |
| 의존 | S1-S3 모두 완료 |
| 외부 의존 | **베타 사용자 모집** (가장 큰 변수) |
| 위험도 | **높음** (외부 시간 의존) |
| 분기 전략 | 모집 더디면 **v1.0.0-rc** 로 분리 출시 후 1.0.1 에서 베타 피드백 반영 |

---

## Goal

`korean-humanizer` 를 v1.0 으로 끌어올린다. 의미는 두 가지:

1. **안정화 약속** — 12 카테고리 / SKILL 출력 포맷 / 정량 규칙 / PROMPT.md API 가 SemVer 룰 안에서 freeze. 사용자가 의존할 수 있다.
2. **외부 검증** — 3-5 명 베타 사용자 1-2 주 운영, 실제 사용 데이터로 룰 미세 조정.

---

## Why

- v0.x 는 "변할 수 있다" 는 신호. 사용자 / contributor 가 본격 의존하기 어려움.
- v1.0 = "이제 부숴지지 않는다" 약속. 마이너 패치는 패턴 추가만, breaking 은 major bump.
- 베타 사용자의 1-2 주 사용 데이터가 현재 examples / 빈도 라벨링이 실제와 맞는지 검증.

#### 8. STABILITY-PROMISE 문서 (0.5d)

`docs/STABILITY-PROMISE.md`:

```markdown
# Stability Promise (v1.0+)

`korean-humanizer` 1.0 부터 SemVer 를 따른다.

## Freeze 영역 (1.x 동안 깨지지 않음)

- **12 카테고리 구조**: 카테고리 추가 / 삭제 / 번호 변경은 major bump (2.0)
- **SKILL.md 출력 포맷**: `## Humanized` + `## 주요 변경 (최대 5개)` 구조 유지
- **정량 규칙**: 20 % cap, 단락 3 곳, 도메인 디폴트 ~다체 금지 룰 유지
- **PROMPT.md API**: 시스템 프롬프트 형식 유지, 사용자가 외부 LLM 에 그대로 붙여 쓸 수 있음
- **카탈로그 4 컬럼**: 헤더 형식 (`나쁨 / 자연스러움 / 빈도 / 적용 도메인`) 유지
- **Personal list Mode A/B/C/D**: 입력 형식 유지

## Minor (1.x.0) 허용 변경

- 카탈로그 패턴 행 추가 / 자연스러움 컬럼 개선
- 새 도메인 추가
- 빈도·도메인 컬럼 재라벨링 (의미는 같지만 더 정확)
- 새 examples 사례 추가
- 문서·README 개선

## Patch (1.0.x) 허용 변경

- 오탈자 / lint 룰 미세 조정
- CI 인프라 변경

## Breaking → 2.0

위 freeze 영역 변경 시. 마이그레이션 가이드 동봉 필수.
```

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| **베타 사용자 모집 실패** (가장 큰 위험) | v1.0.0-rc 로 일단 publish, 1 명만 모이면 진행. 0 명이면 RC → 정식 변환에 1 주 더 사용. |
| 베타 피드백이 carrying 룰 변경 요구 | freeze 약속과 충돌 시 v1.1 으로 미루기. v1.0 은 *현재 룰* 의 안정화. |
| 모집 채널 응답 미흡 | 개인 네트워크 우선 (지인 1-2 명) → 안정화 후 공개 채널 확대 |
| RC 동안 critical bug 발견 | 1.0.0-rc.1 → rc.2 → rc.N 반복 OK. 정식 1.0.0 은 깨끗할 때만. |
| Polish 작업이 시간 부족 | CHANGELOG / SECURITY 는 v1.0.0 동시, MIGRATION 은 v1.0.1 로 미루기 OK |

## Out of scope

- 사용자 만족도 정량 측정 (별도 user study 트랙)
- 마케팅·홍보 (v1.0 release 후 별도 트랙)
- 새 카테고리 추가 (12 freeze, 후속 major)
- 다국어 (영구 out-of-scope)
- 베타 사용자 보상 (외에 README 표기)
- v1.0 이후 후속 (LLM-as-judge / Batch / brand voice 자동 추출 등 — 1.x 마이너로 검토)
