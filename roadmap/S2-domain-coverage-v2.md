# Sprint 2 — v0.7 Domain Coverage v2

> 7 도메인 → 12 도메인. 한국 사용자가 자주 다루지만 카탈로그가 비어 있던 빈자리.

| 항목 | 값 |
|---|---|
| 버전 | v0.6 → v0.7 |
| 기간 | 2 주 (10 working days) |
| 의존 | S1 eval-harness — 신규 사례도 통과해야 함 |
| 외부 의존 | 없음 |
| 위험도 | 낮음 (콘텐츠 작업 위주) |
| 다음 sprint | S3 카탈로그 v2 도메인 컬럼 채우기에 이 도메인 셋 사용 |

---

## Goal

현재 7 도메인 (블로그·마케팅·이메일·LinkedIn·YouTube·뉴스레터·위키) → 12 도메인 확장. 5 개 신규 도메인 사례를 추가하고, 카탈로그에 도메인별 카테고리 우선순위 부록을 신설한다.

---

## Why

- 사용자가 자주 다루는 글쓰기 빈자리 (학술·뉴스·채팅·리뷰·카톡 비즈니스) 가 examples 에 없으면 humanizer 가 어떻게 동작하는지 보여줄 수 없다.
- 카테고리 가중치를 도메인별로 다르게 가져갈 근거 (S3) 를 만들려면 도메인 다양성이 먼저 확보돼야 한다.
- README hero 의 도메인 coverage 가 "12 도메인" 이라고 자신 있게 말하려면 실제 사례 12 개가 필요.

## Tasks

### 1. 도메인 우선순위 결정 (0.5d)

후보 분석:

| 도메인 | 한국 사용자 빈도 | 카탈로그 cat 분포 | 도메인 톤 다양성 | 채택? |
|---|---|---|---|---|
| 학술 초록 / 논문 intro | 중 (대학원생·연구자) | 격식·번역체 (#3) 다수 | 학술 격식 | ✓ |
| 뉴스 기사 | 고 | 수동태 (#6) 다수 | 객관 격식 | ✓ |
| 카톡·DM | 매우 고 | 강조어 (#1) + 이모지 (#10) | informal 친근 | ✓ |
| 제품 리뷰 | 고 (커머스) | 공허한 형용사 (#2) | 솔직 캐주얼 | ✓ |
| 카톡 비즈니스 | 고 (B2B) | 격식·hedging (#3, #11) | ad-hoc 격식 | ✓ |
| X·Threads 포스트 | 고 | 강조어·이모지 | 짧은 캐주얼 | (S3 또는 후속) |
| 강의 소개 | 중 | 발화체 (#9-A) | ~해요체 | (후속) |
| 블로그 결말 | 고 | 판에 박힌 마무리 (#5) | 글말체 | (이미 cover) |

**선정 5 개**: 학술 / 뉴스 / 채팅·DM / 제품 리뷰 / 카톡 비즈니스 메시지.

### 2. 학술 초록 도메인 사례 (1.5d)

`examples/domain-academic.md`:

- 메타 (도메인 톤·강한 카테고리·금지 변경)
- raw: 가상 한국어 논문 abstract (8-10 문장, 격식 + 번역체 + 수동태 짙음)
- humanized (의미 strict 보존, 인용·수치 글자 단위 동일)
- 변경 표 (≤5)
- 보존 영역 (수치, 모델명, 벤치마크명, 인용문 등)
- 학술 도메인 특이 사항: "사료된다 / 시사한다 / 다음과 같다" 같은 격식체는 일부 살림 (학술 톤이 의도된 것)

### 3. 뉴스 기사 도메인 사례 (1.5d)

`examples/domain-news.md`:

- raw: 가상 한국어 뉴스 기사 (단신 6-8 문장, 발생일·인용·기자 의견 분리)
- humanized: 인용문 글자 단위 동일, 사실 strict, 수동태 일부 완화
- 변경 표 + 보존 영역
- 뉴스 특이: 객관 격식이 의도 — 강조어 (#1) 적당히 살림

### 4. 카톡·DM 도메인 사례 (1d)

`examples/domain-chat.md`:

- raw: 카톡 메시지 thread (4-6 turns, AI 가 친근 톤 가장한 흔적 — "정말! 너무 좋네요~")
- humanized: 친근 톤 보존, 강조어 1-2 개만 줄임
- 발화체 도메인 디폴트 ~해요체 / 반말 strict 보존 시연
- 이모지 0-1 개 룰 적용

### 5. 제품 리뷰 도메인 사례 (1d)

`examples/domain-review.md`:

- raw: 가상 커머스 제품 리뷰 (별점 4/5, 4-5 단락, "혁신적인 / 다양한 / 효과적인" 가득)
- humanized: 공허한 형용사 (#2) 정리, 솔직 톤 살림
- 별점·구매일·제품명 strict 보존

### 6. 카톡 비즈니스 메시지 사례 (1d)

`examples/domain-b2b-message.md`:

- raw: 가상 B2B 영업 카톡 (ad-hoc 격식, hedging 짙음 — "확인 가능하실까요?")
- humanized: hedging (#11) 일부 정리, 격식·말줄임 보존
- 비즈니스 카톡 특이: 풀 격식 (~합니다) + 친근 어미 (~할게요) 혼합 OK

### 7. 도메인별 카테고리 우선순위 부록 (1.5d)

`references/ko-ai-signals.md` 부록 E:

```markdown

각 도메인에서 가장 자주 작동하는 카테고리 순위. humanize 시 이 순서로 스캔하면 효율 ↑.

| 도메인 | 1 순위 | 2 순위 | 3 순위 | 톤 보존 (도메인 디폴트) |
|---|---|---|---|---|
| 블로그 (글말체) | #5 개시 | #1 강조어 | #4 것이다 | ~다체 자유 |
| 마케팅 카피 | #2 형용사 | #12 고빈도 어휘 | #1 강조어 | 임팩트 일부 보존 |
| 이메일 (격식) | #11 hedging | #6 수동태 | #3 번역체 | ~합니다체 strict |

| YouTube 인트로 | #5 개시 | #11 hedging | #1 강조어 | ~해요체 (~다체 금지) |
| 뉴스레터 | #5 개시 | #5 마무리 | #7 연결어 | 캐주얼 ~해요체 |
| 위키 / 학술 | #3 번역체 | #6 수동태 | #4 것이다 | 학술 격식 일부 보존 |
```

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| 도메인 raw 콘텐츠 생산이 무거움 (5 개 × 1 시간 이상) | 사례당 raw 분량을 4-6 문장으로 짧게 — long-form.md 가 이미 장문 cover |
| 학술·뉴스 도메인의 사실 보존 검증 어려움 | 가상 데이터지만 수치·고유명사 명시적으로 표시하고 보존 영역 표 첨부 |
| 카톡·DM 의 informal 톤이 humanizer 가 너무 정리해버릴 위험 | 도메인 frontmatter 에 "informal: true" 추가, M4 가 이를 인식 |
| 부록 E 의 우선순위 표가 주관적 | 데이터 수집은 S3 에서 — S2 는 직관 기반 v1 으로 시작, 후속 PR 환영 |
